NBViewer部署

NBViewer

NBViewer提供了Notebook 文件快速的、友好的、快速的查看， 将分析和报告想结合， 做到即使分析也是报告。并支持多种途径查看，最常见的文GitHub。也可在本地提供查看。

名称 版本 功能
Ubuntu系统 16.04及以上
Anaconda 5.3.0及以上 运行环境管理以及依赖安装
python 3.5及以上 运行语音
nodejs 10.8.0 JS依赖

依赖安装

安装Anaconda

Anaconda默认带有python3.6， 所以不再需要安装python。

安装 nodeJS

使用conda 命令安装， 所有依赖会自行安装：

conda install -c conda-forge nodejs

NBViewer 安装

1. 创建NBViewer运行环境

conda create --name nbviewer[虚拟环境名称] python=[python版本]
激活虚拟环境

Linux 激活虚拟环境命令：

source activate [虚拟环境名称]

2. 获取源代码

git clone http://106.15.198.200/sunxr/nbviewer_meta.git

3. NBViewer 安装

进入项目目录下， 执行以下命令安装：

pip install -e .

等待NBViewer 安装完成。

4. 确认安装完成

使用python 启动项目。 确认可正常运行。

python -m nbviewer

主要参数

--localfiles: 开启本地展示，可通过`/localfile`路由访问本地文件。并且是指定NBViewer开始的根目录。

--host: 运行主机IP， 默认为0.0.0.0。

--port: 运行端口，默认为5000。

--debug: 是否开启DEBUG模式， 默认为False。

--localfile_any_user: 是否提供本地系统上其他无法读取的文件（不是notebook文件），默认为False。

后台运行

使用Linux nohup运行NBViewer，携带启动参数请参考上述常用选项：

nohup python -m nbviewer [option] &

示例启动命令：

nohup python -m nbviewer --localfiles=/path/to --port=8888 &

注意点：

部署中有有可能出现依赖缺失，遇的有**invoke**缺失，使用conda命令安装：

conda install -c conda-forge invoke

遇到lesspy缺失， 同样使用conda安装：

conda install -c onda-forge lesspy

执行pip install -e . 报gcc错误：

sudo apt-get install libmemcached-dev |  python-dev |  libssl-dev  |   libcurl4-openssl-dev  |  zlib1g-dev
