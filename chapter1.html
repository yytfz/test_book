JupyterLab

一、概述

JupyterLab 被认为是 Jupyter Notebooks 的进一步发展。其支持更加灵活和更加强大的项目操作方式，但具有和 Jupyter Notebooks 一样的组件。JupyterLab 环境与 Jupyter Notebooks 环境完全一样，但具有生产力更高的体验。

JupyterLab使您能够以灵活，集成和可扩展的方式处理文档和活动，如Jupyter Notebook，文本编辑器，终端和自定义组件。您可以使用选项卡和拆分器在工作区中并排排列多个文档和活动。文档和活动相互集成，实现了交互式计算的新工作流程，例如：

  • 代码控制台 提供瞬态暂存器，用于交互式运行代码，完全支持丰富的输出。 例如，代码控制台可以作为来自笔记本的计算日志链接到笔记本内核。
  • 内核支持的文档 使任何文本文件（Markdown，Python，R，LaTeX等）中的代码能够在任何Jupyter内核中以交互方式运行。
  • 笔记本电脑单元输出可以镜像到自己的选项卡中，与笔记本并排显示，从而实现具有内核支持的交互式控件的简单仪表板。
  • 使用不同编辑者或查看者的多个文档视图可以实时编辑其他查看者中反映的文档。例如，可以轻松实时预览Markdown，分隔符分隔值或Vega / Vega-Lite文档。

JupyterLab还提供了用于查看和处理数据格式的统一模型。JupyterLab了解许多文件格式（图像，CSV，JSON，Markdown，PDF，Vega，Vega-Lite等），并且还可以以这些格式显示丰富的内核输出。有关更多信息，请参阅文件和输出格式。

为了导航用户界面，JupyterLab提供可自定义的键盘快捷键，并能够在文本编辑器中使用来自vim，emacs和Sublime Text的键映射。JupyterLab扩展可以自定义或增强JupyterLab的任何部分，包括新主题，文件编辑器和自定义组件。

JupyterLab从同一台服务器提供服务，并使用与经典Jupyter笔记本相同的笔记本文档格式。

二、安装以及启动

JupyterLab可以使用安装conda，pip。

---1，如果使用conda，可以使用以下命令安装：
conda install -c conda-forge jupyterlab
---2，如果使用pip，可以使用以下命令安装：
pip install jupytelab
---3，如果使用的是早于5.3的Jupyter Notebook版本，则还必须运行以下命令以启用JupyterLab服务器扩展
jupyter serverextension enable --py jupyterlab --sys-prefix

JupyterLab需要Jupyter Notebook 4.3或更高版本。要检查notebook已安装的软件包的版本：

jupyter notebook --version
---5，使用以下命令启动JupyterLab:
jupyter lab

三、页面栏目

  • 菜单栏

file(文件)：与文件和目录相关的操作

edit(编辑)：与编辑文档和其他活动相关的操作

view(视图)：改变JupyterLab外观的操作

run(运行)：用于在不同活动（如笔记本和代码控制台）中运行代码的操作

kernel(内核)：管理内核的操作，内核是运行代码的独立进程

tabs(选项卡)：停靠面板中打开的文档和活动的列表

settings(设置)：常用设置和高级设置编辑器

help(帮助)：JupyterLab和内核帮助链接列表

  • 左侧边栏

files：主要对目录、文件的操作，打开目录、创建文件等。

running：查看当前在运行的核心以及控制台

commands：提供命令搜索功能。

tabs：显示当前打开的文件窗口以及切换功能。
